解决自定义域名失效问题

一、问题
我们在执行 hexo d，部署到 GitHub 仓库时，发现之前设置的自定义域名失效，恢复为默认域名（仓库名），每次更新部署都需要重新设置。

二、解决办法
1、打开配置文件
打开配置文件 _contig.yml 找到“url”，确认此处域名，为之前设置的自定义域名。

2、创建 CNAME 文件
进入“source”文件夹，创建文件“CNAME”，填写自己的域名，“例：wwwanghua.cn”

三、完成
再次执行“hexo d”更新部署，可以正常使用啦！
